# 项目类型与右侧面板实现

## 核心思路

**项目类型体系：**
- `secretary`（秘书）：每个用户有且仅一个，自动创建，类型不可修改。右侧显示"专注模式"面板
- `development`（开发）：用户新建时选择。右侧显示"工作空间"文件浏览面板
- `office`（办公）：用户新建时选择。右侧同样显示"工作空间"文件浏览面板

**数据模型变更：**
- ProjectInfo 新增 `type: 'secretary' | 'development' | 'office'` 字段
- 默认项目改名逻辑：`ensureDefaultProject` 创建时 type 设为 `'secretary'`
- 新建项目对话框增加类型选择（开发/办公）

**面板路由规则：**
- 秘书类型 -> 右侧默认显示 FocusPanel（专注模式，mock 数据）
- 开发/办公类型 -> 右侧可打开 FilesPanel（工作空间，实际文件浏览）
- 所有类型都可使用 skills/agents/channels/settings 面板
